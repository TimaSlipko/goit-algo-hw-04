Емпіричне тестування підтвердило очікування: insertion sort O(n²) стає все більш непрактичним для більших масивів; merge sort O(n log n) задовільний на всіх розмірах. Timsort найефективнігий завдяки гібридному підходу - insertion sort для малих фрагментів (менше за 64 елементи) та merge sort для великих, а також виявляє вже відсортовані послідовності, що вже робить його у декілька разів швидшим за чистий merge sort на випадкових даних та у декілька десятків разів швидшим на відсортованих. Також sorted()/sort() оптимізовані на рівні C.
